Куток · user flows

Як людина проходить шлях

Flows виведені з jobs, не зі структури конкурентів. Кожен показує не лише happy-path: стани empty / loading / error окремо, і обидва кінці — успіх і тупики, де людина застрягає.

бік Ані (Main + RJ1–RJ3) + бік господаря (E) · вузли звірені зі sitemap

екран (зі sitemap) точка рішення стан: empty / loading / error тупик успіх
00

Main Job — безпечно знайти, з ким і де жити

primary · Аня

«Коли я шукаю людину для спільного проживання серед незнайомих, я хочу бути впевненою, що вона реальна, щоб прийняти рішення без страху пошкодувати.» Повний наскрізний flow шукача.

flowchart TD
    Start(["Аня заходить уперше"]) --> Create["Створення профілю"]
    Create --> Feed["Стрічка: кімнати + люди, з фільтром"]

    Feed --> Q1{"Є результати?"}
    Q1 -->|ні| Empty["Стан: нічого не знайшлось"]
    Empty --> Loosen["Дія: послабити фільтр"]
    Loosen --> Feed
    Q1 -->|так| View["Профіль іншої людини / Оголошення кімнати"]

    View --> Q2{"Профіль вселяє довіру?"}
    Q2 -->|ні| Feed
    Q2 -->|так| Q3{"Я верифікована?"}

    Q3 -->|ні| Verify["Верифікація: телефон + селфі"]
    Verify --> Q4{"Верифікація успішна?"}
    Q4 -->|ні| VerErr["Стан: збій — SMS / код / фото / нема камери"]
    VerErr --> Q4b{"Скільки невдач?"}
    Q4b -->|менше N| Verify
    Q4b -->|N і більше| Esc["Ескалація: підтримка; стрічку видно й без верифікації"]
    Esc --> Feed
    Q4 -->|так| Send
    Q3 -->|так| Send["Дія: «Хочу познайомитись»"]

    Send --> Qs{"Заявку надіслано?"}
    Qs -->|збій| NetErr["Стан: помилка мережі"]
    NetErr -->|повторити| Send
    NetErr -->|зберегти і вийти| Feed
    Qs -->|так| Wait["Стан: чекаємо відповідь"]

    Wait --> Q5{"Відповіли на заявку?"}
    Q5 -->|тиша| Stuck["Тупик: відповіді немає"]
    Q5 -->|відхилили| Rej["Стан: заявку відхилено"]
    Q5 -->|так, взаємно| Chat["Чат (текст)"]
    Rej --> Feed
    Stuck --> Feed

    Chat --> Meet["Домовились про зустріч — поза застосунком"]
    Meet --> Success(["Успіх: знайшла, з ким і де жити"])

    classDef state fill:#fff3e0,stroke:#e8a33d,color:#5b4a2f;
    classDef stuck fill:#fde4e1,stroke:#d2553f,color:#5b2a22;
    classDef ok fill:#e7f3ea,stroke:#4a9d6a,color:#234a33;
    class Empty,VerErr,NetErr,Wait,Rej,Esc state;
    class Stuck stuck;
    class Success ok;
Точки зриву Порожня стрічка → послаблення фільтра або відвал (cold-start окремо — RJ1); збій верифікації → після N невдач ескалація (не вічна петля), стрічку видно й без неї; тиша й відмова на заявку — найчастіше. Успіх — поза застосунком (зустріч); продукт відповідає за безпечний шлях до неї і за утримання розмови до зустрічі.
01

RJ1 — знайти серед багатьох тих, хто базово підходить

фільтр · empty

«Коли я починаю шукати серед десятків профілів, я хочу швидко відсіяти тих, хто точно не підходить.» Фокус: фільтр і стан порожнього результату.

flowchart TD
    A["Стрічка: кімнати + люди, з фільтром"] --> B["Дія: фільтр — район, ціна, стать"]
    B --> L["Стан: оновлюємо результати"]
    L --> C{"Є результати?"}

    C -->|ні| W{"Чому порожньо?"}
    W -->|фільтр завузький| E["Стан: під фільтр нічого"]
    W -->|база порожня — cold-start| ZI["Стан: ще немає пропозицій"]
    E --> F{"Послабити фільтр?"}
    F -->|так| B
    F -->|ні| Reset["Скинути фільтр — дивитися все"]
    Reset --> A
    ZI --> ZN["CTA: сусідні райони / повідомити пізніше"]
    ZN --> A

    C -->|так| G["Переглядає картки"]
    G --> H{"Є хоч одна підходяща?"}
    H -->|ні| B
    H -->|так| I["Профіль іншої людини / Оголошення кімнати"]
    I --> Ok(["Далі → RJ2: оцінити довіру"])

    classDef state fill:#fff3e0,stroke:#e8a33d,color:#5b4a2f;
    classDef stuck fill:#fde4e1,stroke:#d2553f,color:#5b2a22;
    classDef ok fill:#e7f3ea,stroke:#4a9d6a,color:#234a33;
    class E,L,ZI,ZN,Reset state;
    class Ok ok;
Точки зриву Два різні «порожньо»: фільтр завузький (→ послабити / скинути) і cold-start, база порожня (→ сусідні райони / повідомити пізніше). Жоден не термінальний тупик; на старті домінує cold-start.
02

RJ2 — переконатися, що людина реальна, до контакту

ядро MVP · trust

«Коли я натрапила на профіль, що здається підходящим, я хочу переконатися, що це реальна людина, щоб не ризикувати наодинці з незнайомцем.» Фокус: бейдж із розшифровкою, повнота, відгуки.

flowchart TD
    A["Стрічка: кімнати + люди, з фільтром"] --> B["Профіль іншої людини"]
    B --> C{"Є бейдж верифікації?"}
    C -->|ні| Stuck1["Тупик: непідтверджений — не ризикує"]
    C -->|так| D["Розшифровка: телефон ✓ / фото ✓ / документ —"]

    D --> E{"Профіль заповнений?"}
    E -->|ні, порожній| Stuck2["Сумнів: повертається у Стрічку"]
    E -->|так| F{"Є відгуки?"}

    F -->|ні| G["Стан: ще немає відгуків — cold start"]
    F -->|так| H["Читає відгуки про людину"]

    G --> I{"Довіри достатньо без відгуків?"}
    H --> I
    I -->|ні| Stuck2
    I -->|так| Trust(["Готова до контакту → RJ3"])

    Stuck1 --> A
    Stuck2 --> A

    classDef state fill:#fff3e0,stroke:#e8a33d,color:#5b4a2f;
    classDef stuck fill:#fde4e1,stroke:#d2553f,color:#5b2a22;
    classDef ok fill:#e7f3ea,stroke:#4a9d6a,color:#234a33;
    class G state;
    class Stuck1,Stuck2 stuck;
    class Trust ok;
Точки зриву Відсутній бейдж = миттєвий тупик (research R1/R2); порожній профіль = сумнів; cold start відгуків — empty-стан, що на старті стосуватиметься майже всіх профілів (тому відгуки = post-MVP).
03

RJ3 — почати розмову тільки якщо інтерес взаємний

ядро MVP · gate

«Коли я вирішила написати першою, я хочу знати, що людина навпроти теж відкрита, щоб не відчувати, що вторгаюсь.» Двосторонній gate: чат відкривається лише при взаємності.

flowchart TD
    A["Профіль іншої людини / Оголошення кімнати"] --> B["Дія: «Хочу познайомитись»"]
    B --> C{"Заявку надіслано?"}
    C -->|збій| Err["Стан: помилка мережі"]
    Err -->|повторити| B
    Err -->|зберегти і вийти| My
    C -->|так| My["Мої заявки: надіслані / очікують"]

    My -->|передумала / «дзвіночки»| Wd["Дія: відкликати заявку"]
    Wd --> A
    My --> Sig["In-app сигнал на вкладці Заявки — не push"]

    Sig --> E{"Вхідні в іншої сторони"}
    E -->|ще немає| EmptyIn["Стан: порожньо — ще немає кандидатів"]
    E -->|є| F{"Інша сторона теж цікавиться?"}

    F -->|ні, відхилила| Rej["Стан: заявку відхилено"]
    F -->|тиша| Silent["Стан: без відповіді — заявка активна"]
    F -->|так, взаємно| G["Чат (текст) — розблоковано"]

    Rej --> Back["Список діалогів"]
    Silent --> Back
    EmptyIn --> Back
    Back -->|шукати далі| A

    G --> H{"Є повідомлення?"}
    H -->|ще немає| GEmpty["Стан: ще немає повідомлень"]
    H -->|так| Ok(["Успіх: розмова почалась"])
    GEmpty --> Ok

    classDef state fill:#fff3e0,stroke:#e8a33d,color:#5b4a2f;
    classDef ok fill:#e7f3ea,stroke:#4a9d6a,color:#234a33;
    class Err,My,Rej,Silent,EmptyIn,GEmpty,Sig,Wd state;
    class Ok ok;
Точки зриву Відмова й тиша — два різні негативні кінці (відхилення видиме, тиша — ні); мала база на старті → мало взаємних інтересів → відчуття «мертвого» продукту (research, патерн №4 «коли ламається»).
04

E — бік господаря: підселити, не впустивши «не ту людину»

Олена · supply

«Коли я підселяю в свій дім, я хочу відфільтрувати кандидатів до зустрічі й переконатися, що людина реальна, щоб не впустити проблему.» Сторона пропозиції: «пошук» — це перегляд кандидатів на оголошення.

flowchart TD
    Start(["Олена заходить"]) --> Onb["Онбординг: профіль + верифікація"]
    Onb --> Create["Створення оголошення: фото, опис, умови"]
    Create --> Pub{"Опубліковано?"}
    Pub -->|збій фото| UpErr["Стан: помилка завантаження"]
    UpErr -->|повторити| Create
    Pub -->|так| Live["Оголошення активне"]

    Live --> Inbox{"Є вхідні заявки?"}
    Inbox -->|ще немає| EmptyC["Стан: ще немає кандидатів — порада зробити помітнішим"]
    EmptyC --> Wait["Чекає — in-app сигнал на нову заявку"]
    Wait --> Inbox
    Inbox -->|є| Cand["Вхідні заявки / кандидати"]

    Cand --> View["Профіль кандидата: бейдж, повнота, відгуки"]
    View --> Q{"Реальний і підходить?"}
    Q -->|ні / дзвіночки| Decline["Відхилити / поскаржитись"]
    Decline --> Cand
    Q -->|так| Accept["Прийняти → взаємно"]
    Accept --> Chat["Чат (текст) — розблоковано"]
    Chat --> Meet["Зустріч — поза застосунком"]
    Meet --> Ok(["Успіх: знайшла сусіда, якому довіряє"])

    classDef state fill:#fff3e0,stroke:#e8a33d,color:#5b4a2f;
    classDef ok fill:#e7f3ea,stroke:#4a9d6a,color:#234a33;
    class UpErr,EmptyC,Wait state;
    class Ok ok;
Точки зриву Аплоуд фото (error/loading); порожні вхідні на cold-start — стан за замовчуванням, тому потрібна порада, а не порожнеча; «реальність» кандидата тримається на тому ж бейджі з розшифровкою (RJ2 з боку господаря).
